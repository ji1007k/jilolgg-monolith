# 파티션/동시성 튜닝 단계별 가이드

복잡한 공식보다, 아래 순서대로 작은 폭으로 조정하는 방식이 안전합니다.

## 0. 현재 기준값 고정
- 시작값: `gridSize=5`
- 한 번에 하나만 바꿉니다. (`gridSize`, executor, DB pool 동시 변경 금지)

## 1. 먼저 볼 지표 4개
- 배치 총 소요시간
- DB 커넥션 상태 (active / pending)
- 외부 API 오류율 (특히 429, timeout)
- 서버 CPU/메모리

## 2. 변경 단위
- 작은 폭으로만 증가: 예) `5 -> 6 -> 8`
- 각 단계는 최소 1~2일 관찰

## 3. 중단/롤백 기준
- 429 또는 timeout이 눈에 띄게 증가하면 즉시 이전 값으로 롤백
- DB pending connection이 지속적으로 쌓이면 즉시 롤백
- API p95 응답시간이 기준치를 초과하면 롤백

## 4. 확정 기준
- 배치 시간 개선 + 오류율 안정, 두 조건을 모두 만족한 마지막 값을 운영값으로 확정

## 5. 실무 팁
- 트래픽이 급증하는 시간대에는 튜닝하지 않기
- 운영 적용 전 동일 데이터셋으로 최소 2회 이상 재현 확인
- 튜닝 기록(변경값/시간/지표/결론)을 문서로 남기기

## 한 줄 요약
“작게 올리고, 지표 보고, 이상하면 바로 되돌리는 방식으로 튜닝한다.”
