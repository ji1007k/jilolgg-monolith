## **개선 방향**

- **Docker Compose 기반 통합 배포 구성**
    - 백엔드, 프론트엔드, Redis, DB 등 전체 스택 통합
    - 로그 파일 외부 볼륨 마운트 포함

- **경기 일정 관리 자동화**
    - 일/주/월 단위로 경기 일정 자동 갱신
    - 기존 일정과 비교해 변경 감지 후 자동 업데이트
 
- **관리자 기능 구현**
    - 리그/토너먼트/팀/경기일정 수동 관리 기능 구현 (API 데이터 지연 대응)

- **실시간 채팅 고도화**
    - Pub/Sub 구조는 메시지 유실에 취약하므로, 향후 Redis Streams로 전환해 메시지 지속성 및 처리 순서를 보장할 계획
    - 추후 DB로 백업 또는 분석 기반 활용 고려

- **경기별 커뮤니티 기능 추가**
    - 경기별 채팅방 개설 (현재 채팅방 1개만 존재)
    - 경기별 다이어리 기능 도입 (자유 게시글/응원 글 작성 등)

- **소셜 로그인**
    - OAuth2 기반 소셜 로그인

- **Redis 기반 인증 정보 캐싱**
    - 소셜 로그인 인증 코드, 사용자 정보 캐시 처리
    - 멀티 디바이스 로그인 관리 및 로그아웃 처리 개선
        - 현재는 브라우저 쿠키 삭제 방식만 사용
        - 모든 기기에서 로그아웃 기능 별도 구현 예정
        - 비밀번호 변경 시 자동 전체 로그아웃 기능 포함

- **Riot API 추가 연동**
  - 공식 경기 외 일반 사용자 인게임 데이터 수집
  - 전적 검색 또는 커뮤니티 기능 연계 목적
