## 🔧 Git 관리

### 기본 Clone
```bash
# 현재 디렉토리에 바로 클론 (하위 폴더 생성 안함)
git clone https://github.com/ji1007k/basic-fe-nextjs.git .
```

### Git Submodule 사용
```bash
# 서브모듈 추가
git submodule add <be 레포 URL> <폴더명>
git submodule add <fe 레포 URL> <폴더명>

# 서브모듈 초기화 (폴더 생성, 하위 서브모듈까지 초기화)
git submodule update --init --recursive

# 최신 서브모듈 커밋으로 갱신
git submodule update --remote --merge
```

### 커밋 취소 방법

#### 방법1: 강제 리셋 (혼자 작업 시만)
1. IntelliJ에서 Git → Log 탭 열기
2. 취소하고 싶은 커밋 이전 커밋을 우클릭 → Reset Current Branch to Here...
3. Hard 선택 (파일도 커밋 전 상태로 되돌림)
4. 강제 푸시: `git push origin HEAD --force`

#### 방법2: Revert 커밋 (협업 시 추천)
1. Git → Log 탭 열기
2. 되돌리고 싶은 커밋을 우클릭 → Revert Commit
3. 자동으로 "되돌리는 커밋" 생성
4. 일반 Push
⚠️ **주의**: 수정사항이 사라집니다

## 🐳 Docker 관리

### 기본 명령어
```bash
# 시스템 정리
docker system prune -a
docker system df -a

# 컨테이너 빌드 및 실행
docker compose up -d --build
docker compose build be/fe
docker compose up -d be/fe

# 특정 compose 파일 사용
docker compose -f compose.yaml up
docker compose -f composeA.yaml -f composeB.yaml up

# 컨테이너 내부 접근
docker exec -it 컨테이너명 sh
```

### 메모리 모니터링
```bash
# 컨테이너 메모리 사용량 확인
docker stats

# 컨테이너 내부에서 JVM 모니터링
docker exec -u 0 -it [컨테이너명] sh
apk add openjdk17
jps                           # PID 확인
jstat -gc [pid] 1000         # 1초마다 GC 상태 모니터링
```

## 🗄️ Redis 관리

### 접속 및 기본 명령어
```bash
# Redis CLI 접속
redis-cli -h 54.180.118.74 -p 36379 -a jikim_pwdForRedis_250419

# 모든 데이터 삭제
FLUSHALL

# 메모리 정보 확인
info memory

# 전체 키 목록 확인
KEYS *                       # 모든 키 (부하 위험)
SCAN 0                       # 커서 기반 일부 키 (권장)
```

### Redis 메모리 설정

#### 방법1: 설정파일 수정 (영구적)
```bash
# 설정파일 위치 찾기
sudo find / -name redis.conf

# redis.conf 내용
maxmemory 128mb
maxmemory-policy allkeys-lru
```

#### 방법2: 런타임 변경 (임시적)
```bash
redis-cli -h 127.0.0.1 -p 36379 -a jikim_pwdForRedis_250419 config set maxmemory 128mb
redis-cli -h 127.0.0.1 -p 36379 -a jikim_pwdForRedis_250419 config set maxmemory-policy allkeys-lru
```

## 🖥️ 시스템 관리

### 포트 관리
```bash
# 특정 포트 사용 프로세스 확인
netstat -ona | findstr PORT

# 프로세스 강제 종료
taskkill /f /pid PROCESS_ID
```

### 시스템 서비스 관리
```bash
sudo systemctl status [서비스명]    # 상태 확인
sudo systemctl start [서비스명]     # 시작
sudo systemctl restart [서비스명]   # 재시작
sudo systemctl stop [서비스명]      # 중지
sudo systemctl enable [서비스명]    # 부팅시 자동 시작 설정
sudo systemctl disable [서비스명]   # 부팅시 자동 시작 해제
sudo systemctl is-enabled [서비스명] # 자동 시작 상태 확인
```
