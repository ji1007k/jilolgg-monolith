# 📝 Git 커밋 메시지 컨벤션

이 문서는 프로젝트 내에서 일관된 커밋 메시지를 작성하기 위한 가이드입니다.

---

| 라벨        | 설명                                                  |
|-------------|-------------------------------------------------------|
| `feat`      | 새로운 기능 추가 또는 구조적 개선                             |
| `fix`       | 버그 수정 또는 기존 기능의 잘못된 동작 수정                      |
| `refactor`  | 코드 리팩토링 (기능 변경 없이 구조만 개선)                    |
| `chore`     | 빌드 설정, 환경 구성, 배치 작업, 테스트 데이터 등 개발 보조 작업      |
| `test`      | 테스트 코드 추가, 수정, 리팩토링 등 테스트 관련 작업               |
| `docs`      | 문서 파일(`.md`, 주석 등)의 추가 또는 수정                      |
